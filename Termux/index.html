Manipulação de Diretórios e Arquivos

  • ls: Lista os arquivos e diretórios.
  • ls-a: Lista todos arquivos, (incluindo arquivos ocultos que começam com .).
  • ls-l: Lista arquivos com detalhes, ( permissões, tamanho, etc. )
  • cd: Navega até um diretório.
  • cd ..: Volta um duretório.
  • cd/: Volta para o diretório raiz.
  • cd: Volta ao diretório inicial.
  • rm: Remove arquivos ou diretórios.
    Exemplos: 
  • rm (arqiivo.txt)
    Remove arquivo txt com ou sem conteúdo.
  • rmdir (diretório)
    Remove diretório vazio.
  • rm -r (diretório)
    Remove diretório e seus conteúdos.
  • rm -rf (diretório)
    Remove diretório e seus conteúdos sem confirmação.
  • mv: Move ou renomeia arquivos e diretórios.
    Exemplos: 
  • mv (nome_antigo_arquivo) (novo_nome_arquivo)
    Renomeia o arquivo.
  • mv (nome_antigo_diretório) (novo_nome_diretório)
    Renomeia diretório.
  • mv (arquivo ou diretório) diretório/
    Move o arquivo ou diretório, para um subdiretório, no diretório atual.
  • mv (arquivo ou diretório) /caminho/do/destino/
    Move um arquivo ou diretório para outro diretório diferente do atual.
  • mv /caminho/de/origem/arquivo .
    Move um arquivo de outro diretório para o diretório atual.
  • mv /caminho/de/origem/Pasta ./
    Move uma pasta inteira de outro diretório para o diretório atual.
  • mv /caminho/de/origem/arquivo_ou_diretório caminho/de/destino
    Mover um arquivo ou diretório que nem a origem nem o destino seja o diretório atual.
  • cp: Copia arquivos ou diretórios.
    Exemplos: 
  • cp (arquivo ou diretório) diretório/
    Copia o arquivo ou diretório, para um subdiretório, no diretório atual.
  • cp (arquivo ou diretório) /caminho/do/destino/
    Copia um arquivo ou diretório para outro diretório fora do atual.
  •  cp (diretório com subdiretórios e arquivos) /caminho/do/destino
    Copia um diretório juntoncom subdiretórios e arquivos para outro diretório fora do atual.
  • echo: Exibe texto ou adiciona conteúdo a um arquivo.
    Exemplos: 
  • echo "Olá, Termux!"
    Exibe a mensagem "Olá, Termux!" no terminal.
  • echo "Bem-vindo" > arquivo.txt
    Cria o arquivo arquivo.txt com o texto "Bem-vindo".
  • echo "Nova linha" >> arquivo.txt
    Adiciona "Nova linha" ao final do arquivo arquivo.txt.
  • echo -e "1. Item\n2. Item\n3. Item" > lista.txt
    Cria o arquivo lista.txt com uma lista numerada.
  • echo -e "• Item 1\n• Item 2\n• Item 3" > lista.txt
    Cria o arquivo lista.txt com itens pontuados.
    • cat: Exibe o conteúdo de um arquivo ou combina arquivos.
    Exemplos: 
  • cat arquivo.txt
    Exibe o conteúdo do arquivo arquivo.txt no terminal.
  • cat arquivo1.txt arquivo2.txt > combinado.txt
    Combina os conteúdos de arquivo1.txt e arquivo2.txt em um novo arquivo chamado combinado.txt.
  • cat -n arquivo.txt
    Exibe o conteúdo de arquivo.txt com os números das linhas.

Comandos de Rede

  • ifconfig: Exibe informações de rede.
  • curl https://api.ipify.org: Exibe informações do IP Público, atribuído pelo provedor de Internet.
  • ping: Testa a conectividade com outro dispositivo.
  • wget: Faz o download de arquivos da internet.

Tutorial: Instalar e Configurar Apache

  1. Caminho padrão do Apache2 no Termux 
    /data/data/com.termux/files/usr/share/apache2/default-site/htdocs
  2. Instalar o Apache: 
    pkg install apache2
  3. Iniciar o Apache: 
    apachectl start
  4. Verificar status do Apache: 
    apachectl status
  5. Parar o Apache: 
    apachectl stop

Comandos de Ferramentas

Comando nmap

O nmap é uma ferramenta poderosa para escanear redes e obter informações sobre hosts e serviços.

Exemplo de uso básico:

nmap 192.168.1.0/24

Este comando escaneia todos os dispositivos na rede 192.168.1.0/24.

Outros exemplos:

  • nmap -sS 192.168.1.1
    Escaneia portas TCP em modo stealth.
  • nmap -O 192.168.1.1
    Detecta o sistema operacional.
  • nmap --script=vuln 192.168.1.1
    Busca vulnerabilidades.

Comando curl

O curl é usado para realizar requisições HTTP diretamente do terminal.

Exemplo:

curl https://example.com

Este comando faz uma requisição à página https://example.com e exibe a resposta no terminal.

Comandos de Permissões

Comando chmod

O comando chmod é usado para alterar as permissões de arquivos e diretórios no Linux.

Entendendo os Números de Permissões

As permissões são divididas em três partes:

  • Dono: O proprietário do arquivo.
  • Grupo: Os usuários pertencentes ao mesmo grupo do arquivo.
  • Outros: Todos os outros usuários.

Exemplos de Permissões Comuns:

  • 644:
    • -rw-r--r--
    • Dono: Leitura e Escrita.
    • Grupo: Apenas Leitura.
    • Outros: Apenas Leitura.
  • 777:
    • -rwxrwxrwx
    • Dono: Leitura, Escrita e Execução.
    • Grupo: Leitura, Escrita e Execução.
    • Outros: Leitura, Escrita e Execução.
  • 600:
    • -rw-------
    • Dono: Leitura e Escrita.
    • Grupo: Sem Permissão.
    • Outros: Sem Permissão.
  • 755:
    • -rwxr-xr-x
    • Dono: Leitura, Escrita e Execução.
    • Grupo: Leitura e Execução.
    • Outros: Leitura e Execução.

Comandos:

  • chmod 644 arquivo.txt
    Define permissão de leitura para todos, mas apenas o dono pode escrever.
  • chmod 777 arquivo.txt
    Dá acesso total (leitura, escrita e execução) para todos.
  • chmod 600 arquivo.txt
    Permite acesso apenas ao dono (leitura e escrita).
  • chmod 755 arquivo.txt
    Permite acesso total ao dono e somente leitura/execução para outros.

Use as permissões com cuidado, especialmente 777, pois pode expor arquivos a riscos de segurança.

Comando chown

O comando chown altera o proprietário ou grupo de arquivos.

Exemplo:

chown usuario:grupo arquivo.txt

Entendendo as Permissões

As permissões são representadas como:


-rwxr-xr--
Dono: rwx (7)
Grupo: r-x (5)
Outros: r-- (4)

Significado:

  • r: Leitura
  • w: Escrita
  • x: Execução
  • -: Sem permissão
